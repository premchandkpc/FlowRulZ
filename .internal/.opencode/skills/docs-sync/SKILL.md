# Docs Sync

Use this skill after code changes that affect architecture, behavior, interfaces, or developer workflows.

## When to use
- New features, refactors, or bug fixes change runtime behavior
- Public interfaces or internal modules change
- The repository instructions require documentation updates

## Responsibilities
- Update relevant markdown files in the docs tree
- Keep architecture notes, developer guides, and feature docs aligned with the actual implementation
- Preserve the repository's documentation conventions and terminology

## Output expectations
- Identify affected docs files
- Update them with concise, accurate summaries
- Avoid stale documentation and keep examples consistent with the code
